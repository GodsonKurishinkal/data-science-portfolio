Quick Start

Calculate price elasticity and optimize revenue in just a few lines of code

elasticity_analysis.py Python 
from src.pricing.elasticity_analyzer import ElasticityAnalyzer

# Initialize analyzer
analyzer = ElasticityAnalyzer(df)

# Log-log regression (most robust)
log_log = analyzer.calculate_elasticity(
    method='log-log',
    price_col='price',
    quantity_col='quantity'
)

print(f"Elasticity: {log_log['coefficient']:.2f}")
print(f"95% CI: {log_log['ci_lower']:.2f} to {log_log['ci_upper']:.2f}")
print(f"P-value: {log_log['p_value']:.4f}")

# Revenue optimization
from src.pricing.price_optimizer import PriceOptimizer

optimizer = PriceOptimizer(
    elasticity=log_log['coefficient'],
    current_price=19.99,
    margin_floor=0.20
)
optimal = optimizer.find_optimal_price()
print(f"Optimal Price: ${optimal:.2f}")

Key Insights

Strategic learnings from analyzing price sensitivity across product categories

Category Variation

Elasticity varies significantly by category. FOODS: -0.8 (inelastic), HOBBIES: -2.3 (highly elastic).

Timing Matters

Elasticity changes with season and promotional context. Holiday periods show 20-30% lower sensitivity.

Segment Response

Budget-conscious segments show 2x higher elasticity than premium segments on same products.

Competitive Context

Elasticity increases when competitors are price-aggressive. Monitor relative positioning.
